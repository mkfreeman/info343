Summary

Over the past two lectures, we've discussed the power of generating web content using an underlying data structure with the angular.js framework. We also reviewed the how to access request data from publicly accessible APIs using AJAX requests.

During this challenge, you will build a web application with Angular that accesses data from the Spotify API. This assignment is deliberately unstructured to encourage you to push the limits of your skills gained thus far in creative ways. The example code provided in class should provide you with the underlying structure for building you app, and allow you to think creatively about what you can do with the data.

Objectives

By completing this challenge, you will practice and demonstrate mastery of the following skills:

  • Use Angular directives and expressions in HTML
  • Create Angular modules and controllers in your JavaScript code.
  • Use an API and Angular's $http service to get data from a web application.

Use the API to get Data

Based on the form input from your user, you should make a request to the Spotify API. The API has a number of different configurations to specify your search. Angular also provides a built in $http service that allows you to make AJAX requests within your application.

Note: in order to leverage the $http service, you must include it as a variable when you create a controller

var myCtrl = myApp.controller('myCtrl', function($scope, $http) {
  $scope.getData = function(url) {
    $http.get(url).success(function(response){
      $scope.data = response.data 
    })
  }
})

Use Angular directives to display data

One of the core strengths of using Angular in your application is the ability to bind underlying data to visual elements. The data retrieved from your $http service should be directly linked to your visual display. If the data changes given a search, you visual elements should automatically update.

  <li ng-repeat="song in songs">
    {{ song.title + ', ' + song.artist }}
  </li>

Note: displaying data goes far beyond showing text on the page. The Spotify API returns images, relationships between artists, and other information that can be integrated into multiple components of your layout.

Do something creative!

Have fun with this challenge! The template was provided to allow you to explore different options of what you can do with the rich dataset. This example is far beyond what I would expect, but hopefully provides inspiration about the types of relationships there are in the data that can be explored. When thinking about how to move forward from the template, consider the following:

  • The Spotify API provides lots of data for each object, including interesting information as well as album art you can display
  • You can (should?) change the data you are requesting using different API endpoints
  • Based on what data you need, you make have to make multiple requests
  • Don't forget about your CRUD data skills with Parse. By saving user data, you could allow users to create custom playlists, compete against one another in a guesing game, or anything else!
