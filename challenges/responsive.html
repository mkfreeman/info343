One of the most common web development projects is a homepage for an organization or product. Most every organization and product has one, and it's amazing how many of them are really terrible, especially on small mobile device screens. This is a shame because it's just not that hard to build a well-designed, responsive marketing site. In this challenge you will build one of these sites for a fictitious coffee shop named Dawg Coffee. This site will respond to the size of screen it is rendered on, and adjust its layout and styling accordingly.

Objectives

The purpose of this challenge is to expose you to many of the built-in functionalities associated with HTML elements and CSS styles. While making this volume of specifications may seem tedious, it is an opportunity to explore the fun and flexible components of web development. By completing this challenge you will practice and demonstrate the following skills

  • Writing semantically-rich HTML pages by choosing the most appropriate elements to wrap around your information
  • Writing well-formatted HTML source that is properly-indented and easy to read
  • Creating hyperlinks that tie pages together
  • Using HTML character entities to represent reserved or difficult to input characters
  • Creating a stylesheet and linking it to your web pages
  • Writing style rules to control the formatting of your pages, including fonts, colors, links, backgrounds, borders, padding, margins, and layouts
  • Using various CSS units, including percentages, pixels, ems, hex colors, and rgba() colors
  • Adding the Bootstrap CSS framework to your web pages to automatically improve the graphic design
  • Using the Bootstrap grid system to create multi-column page layouts
  • Using media queries to adjust page layout and other formatting based on the screen size
  • Implementing a page based on screenshots you would get from a graphic designer

Basic Page Structure

Open index.html in your editor and add the elements for a basic web page. Remember to include the following:

  • The HTML5 doctype declaration.
  • An element telling the browser that this page is saved in the UTF-8 text encoding.
  • An element telling the browser to set the viewport width to the device width, and the initial scaling to 1. This will stop the browser from zooming out to show the entire page on a small mobile screen.
  • An element declaring the page title to be Dawg Coffee
  • An element declaring that the page's icon is the page-icon.png located in the img subdirectory.
  • An element that adds the main.css stylesheet in the css subdirectory to the page.

Now that you have the basic skeleton, you can go on to create and style each portion of the page.

Base Page Styles

You will be building this page in a "mobile first" approach. The base style rules will be the ones that should apply to small phone screens, and you will then add media queries to change those rules for larger screens.

In your main.css file, add the following:

  • A directive to import the Bootstrap CSS file from its content delivery network (CDN) location https://maxcdn.bootstrapcdn.com/bootstrap/3.3.5/css/bootstrap.min.css. You only need the CSS file, not the JavaScript file.
  • Directives to import the Google fonts Raleway and Lato. You will need the 300 and 400 weight normal face of both, and the 300-weight italic face of Lato.
  • A directive to import the Font Awesome font from it's CDN location https://maxcdn.bootstrapcdn.com/font-awesome/4.4.0/css/font-awesome.min.css.

Always use https:// links when referencing resources that live on other servers. If your page is loaded from a secured https:// URL, the browser may refuse to load other assets that come from unsecured http:// urls. Although accessing an https:// link is slightly slower, it's much more secure and is typically necessary if your page ever ends up being served over an HTTPS connection.

Then add some style rules that accomplish the following formatting:

  • The default font should be Lato, with Helvetica Neue, Helvetica, and sans-serif as the fall-backs if the browser cannot reach the Google fonts server.
  • The default font size should be 16 pixels tall, but should increase to 20 pixels on screens 768 pixels and wider.
  • The default font weight should be 300.
  • The background color for the page should be #E9EAEE.
  • All headings should use the font Raleway, with Helvetica Neue, Helvetica, and sans-serif as the fall-backs.
  • All second-level headings should be in the color #BC6026.
  • On screens 768 pixels and wider, all second-level headings should be center aligned, and have a blank space below them equal to the height of the current font.
  • All <svg> elements should be the same height as the current font.

Now create style rules in your main.css file to accomplish the following

Generate Your CSS Using Sass

You may have noticed several things that were repeated in a few different style rules, especially color values. In larger projects this repetition can become extreme, and changing your colors can become quite a chore. To make this more tolerable, most professional developers use a CSS preprocessor tool, the most popular of which is known as Sass.

For this step, rewrite your main.css file in the SCSS language, taking advantage of variables, nesting, and functions. Then use the sass command line tool, to generate your main.css file from your main.scss file. While you're working on this, you will probably want to use their feature that watches your source SCSS file and automatically regenerates the target CSS whenever the source SCSS file changes.

Use variables for all your colors, but for the darker background on the cafe info and story sections, use the Sass darken() function to darken the default page background color by 5%. For the footer background color, darken the default page background color by 20%.

Also use a variable for the set of fall-back fonts that are used if Lato and Raleway cannot be downloaded from the Google fonts server.

Note that Sass will create a hidden directory name .sass-cache in your project that should not be added to your repository. To keep this out of your repository, remember to add .sass-cache to your .gitignore file!

Add the Medium Scroll Effect to the Header Image

You must know a little JavaScript and/or jQuery to complete this step! If you don't know JavaScript yet, don't worry—we will learn it later in class. But if you already know some JavaScript and/or jQuery, give this a shot. It's a very cool effect.

You may have seen that cool blur and fade-out effect that Medium.com made popular: as you start to scroll down the page, the background image in the header begins to darken and blur, in proportion to the amount you scroll. If you already know a little JavaScript, try to recreate that effect in this solution. This can be enabled in one of two ways.

Option 1: Add a Blurred Version of the Background Image and Fade Between with CSS Opacity

See the tutorial "Recreating Medium's Paralax Blur Effect" for a description of this approach. It uses a blurred version of the same background image and simply fades the blurred one in over the other. To do this approach, you will need to use a graphics tool to create blurred versions of the various background images used in this challenge.

Option 2: Use CSS Filters

Many browsers have now added native support for various kinds of graphics filters, including brightness and blur. All the recent browsers support this filter CSS property, but Chrome and Safari currently require you to use the vendor-prefixed version -webkit-filter, because it isn't yet ratified as a standard.

Using CSS filters means that you can darken and blur the existing background image dynamically. You don't need to make blurred versions of the background image ahead of time. Instead, the browser will dynamically apply a darkening and blurring filter in response to your code changing the element's style.filter and style.webkitFilter properties. You can apply multiple filters by setting this property to a list of filter functions separate by spaces. In a CSS rule it looks like this:


-webkit-filter: brightness(50%) blur(10px); /* for Chrome */
filter: brightness(50%) blur(10px);         /* for others */

And in JavaScript it looks like this:


var filter = 'brightness(50%) blur(10px)';
var elem = //...get the element using one of the various methods
elem.style.webkitFilter = filter;       //for Chrome
elem.style.filter = filter;             //for others

The basic technique is to add an event listener for the window object's 'scroll' event and adjust the style.filter and style.webkitFilter values based on the window.pageYOffset value. Remember that window.pageYOffset returns the number of pixels you've scrolled down the screen, but the brightness() filter accepts a percentage, and the blur() filter accepts a number of pixels by which to blur.
